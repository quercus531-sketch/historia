# Historia — сайт учителя истории

Статичный сайт (HTML + CSS + JS, без фреймворков и backend), готовый к размещению на **GitHub Pages**.
Админ-панель работает полностью в браузере, данные хранятся в `localStorage`.

## 🚀 Как загрузить на GitHub и включить GitHub Pages

1. **Создайте репозиторий** на [github.com](https://github.com) (например, `historia` или `my-history-site`). Сделайте его публичным.

2. **Загрузите файлы**. Два способа:

   **Через веб-интерфейс:** на странице пустого репозитория нажмите *"uploading an existing file"* → перетащите все файлы и папки → *Commit changes*.

   **Через терминал:**
   ```bash
   cd путь/к/historia
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/historia.git
   git push -u origin main
   ```

3. **Включите GitHub Pages:**
   - В репозитории откройте **Settings → Pages**
   - В блоке *Build and deployment* выберите **Source: Deploy from a branch**
   - Branch: **main**, папка: **/ (root)** → **Save**
   - Через 1–2 минуты сайт будет доступен по адресу:
     `https://ВАШ_ЛОГИН.github.io/historia/`

Все ссылки в проекте **относительные** — сайт корректно работает как в подпапке, так и в корне.

## 🔐 Админ-панель

### Вход
- В правом нижнем углу любой страницы есть круглая кнопка **✦**. Нажмите её.
- Введите пароль: **`admin123`**
- После входа появится бейдж *«◆ Режим редактора»* и кнопки *+/✎/×* на всех карточках.

### Что можно делать
- **Добавлять, редактировать и удалять:**
  - Учебные материалы (название, эпоха, описание, содержание)
  - Изображения галереи (название, описание, URL картинки)
  - Тесты (название, описание, вопросы в формате JSON)
  - Презентации (PDF-ссылка или iframe Google Slides / OneDrive)
- Нажмите на пустую карточку с «**+**» чтобы добавить новый элемент.
- Кнопка **✎** — редактировать, **×** — удалить.
- Чтобы выйти — снова нажмите кнопку админ-панели.

### Сменить пароль
Откройте `js/script.js`, первая строка:
```js
const ADMIN_PASSWORD = 'admin123';
```
Замените на свой пароль.

### ⚠️ Важно про localStorage
- Данные хранятся **в браузере пользователя**, а не на сервере.
- Это значит: контент, который вы добавили на своём компьютере, **не увидит посетитель** — он увидит только стартовые данные, заложенные в `js/script.js` (`DefaultData`).
- Чтобы «опубликовать» новые материалы для всех — отредактируйте объект `DefaultData` прямо в `js/script.js` и сделайте `git push`. Можно использовать админку как предпросмотр, а затем скопировать JSON из DevTools (`Application → Local Storage`) в код.

## 🎨 Дизайн

- **Палитра:** `#825332` (охра), `#325454` (тёмно-изумруд), бежевый, золотые акценты
- **Фон:** многослойные SVG-текстуры пергамента, карты, шум, затемнение — всё на чистом CSS
- **Шрифты:** Cormorant Garamond (заголовки), PT Serif (текст), Marck Script (рукописные акценты) — подгружаются из Google Fonts, полная поддержка кириллицы
- **Анимации:** плавные появления карточек, hover-эффекты, переходы модалок
- **Адаптивность:** Flexbox + Grid, бургер-меню для мобильных

## 🖼 Как добавить фото учителя

Положите фото в папку `assets/` (например, `assets/teacher.jpg`) и замените в `index.html` блок:

```html
<div class="portrait-frame">
  <div class="portrait-placeholder">A</div>
</div>
```

на:

```html
<div class="portrait-frame">
  <img src="assets/teacher.jpg" alt="Анна Петровна Соколова">
</div>
```
